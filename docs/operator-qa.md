## 1. ロール（権限）について

ユーザーには4種類のロールがあります。操作できる機能がロールによって異なります。


| ロール        | 日本語 | 主な権限                                 |
| ---------- | --- | ------------------------------------ |
| `admin`    | 管理者 | 全機能。アカウント追加/削除、チーム管理、ガードレール設定、監査ログ閲覧 |
| `approver` | 承認者 | 投稿の作成・承認・差し戻し                        |
| `editor`   | 編集者 | 投稿の作成・編集・削除（承認不可）                    |
| `viewer`   | 閲覧者 | 全ページ閲覧のみ（投稿・操作不可）                    |


### よくある質問

**Q. 「承認」ボタンが表示されない**
→ ロールが `editor` または `viewer` の場合、承認ボタンは表示されません。承認を行うには `admin` または `approver` ロールが必要です。チームの管理者に権限変更を依頼してください（→ [チーム管理](#10-チーム管理)）。

**Q. 特定のページに入れない・リダイレクトされる**
→ 以下のページは `admin` ロールのみアクセス可能です。アクセスすると自動的にアカウント管理ページへ移動します。

- 監査ログ（`/audit`）
- チーム管理（`/team`）

## 2. アカウント管理

**画面**: サイドバー →「アカウント管理」

### 2-1. Xアカウントを追加する

> **必要なロール**: `admin`

1. 「アカウント管理」画面を開く
2. 右上の **「アカウントを追加」** ボタンをクリック
3. X（Twitter）の認証画面にリダイレクトされます
4. X のログイン情報を入力し、アプリの連携を許可する
5. 「**アカウントを接続しました**」のメッセージが表示されれば完了

> **注意**: 「アカウントを追加」ボタンは `admin` ロールのみ表示されます。

### 2-2. アカウントの状態を確認する

画面上部のサマリーカードで以下を確認できます。


| 項目      | 説明               |
| ------- | ---------------- |
| 総フォロワー  | 全アカウントのフォロワー合計   |
| アクティブ   | 現在稼働中のアカウント数     |
| 未返信     | 未対応の受信箱アイテム数     |
| 要トークン更新 | トークンが期限切れのアカウント数 |


「**要トークン更新**」が 1 以上の場合、その数字にマウスをホバーすると、対象アカウント名・@ハンドル・期限切れ日が表示されます。

### 2-3. トークンが期限切れになったアカウントを再接続する

1. 「要トークン更新」のホバーで対象アカウントを確認
2. **「アカウントを追加」** ボタンから同じXアカウントで再認証
3. 既存のアカウントデータは保持されたまま、トークンのみ更新されます

## 3. 投稿作成

**画面**: サイドバー → 「投稿作成」（✏️）

> **必要なロール**: `editor` 以上

### 3-1. 投稿を作成して承認依頼する（通常フロー）

1. 「投稿作成」画面を開く
2. **配信先アカウント** を選択（複数選択可）
3. **投稿テキスト** を入力（最大 280 文字）
  - 複数アカウントを選択した場合、「アカウント別に文面を変更」のチェックを入れるとアカウントごとに異なるテキストを設定できます
4. 必要に応じて **予約日時** を設定（任意）
5. 必要に応じて **キャンペーン** を選択（任意）
6. ボタンを選択して送信：
  - **「下書き保存」**: 保存のみ（承認不要）
  - **「承認依頼」**: 承認者に承認依頼を送る
  - **「予約する」**: 予約日時を設定した場合のみ表示。承認後、指定日時に自動投稿

### 3-2. 投稿ステータスの流れ

```
下書き(draft)
  └─「承認依頼」→ 承認待ち(pending)
                    └─「承認」→ 予約あり → 予約中(scheduled) → [指定時刻に自動投稿] → 投稿済み(published)
                              → 予約なし → 承認済み(approved) → [即時投稿] → 投稿済み(published)
                    └─「差し戻し」→ 差し戻し(rejected) → 編集して再提出
```

### 3-3. よくある質問

**Q. 「配信先アカウントを選択してください」と表示される**
→ 投稿先のアカウントが選択されていません。「配信先」エリアのアカウント一覧からチェックを入れてください。アカウントが一覧に表示されない場合、管理者にアカウント追加を依頼してください。

**Q. 「投稿テキストを入力してください」と表示される**
→ テキスト欄が空白のままです。投稿内容を入力してから送信してください。

**Q. ガードレール違反のエラーが表示された**
→ 投稿内容が組織のルールに違反しています（→ [ガードレール](#6-ガードレール)参照）。表示されたメッセージに従って内容を修正してください。

- 例：「NGワード「〇〇」が含まれています」→ 対象の単語を削除・変更
- 例：「深夜帯（23時〜6時）の投稿は禁止されています」→ 予約時間を変更
- 例：「URLにUTMパラメータが付与されていません」→ URLに `?utm_source=...` を追加

**Q. 「予約する」ボタンが表示されない**
→ 予約日時が設定されていないと表示されません。「予約・設定」欄の日時入力フィールドに日時を入力してください。

## 4. 承認ワークフロー

**画面**: サイドバー → 「承認」

> **承認・差し戻し操作**: `admin` または `approver` ロールのみ

### 4-1. 承認待ちの投稿を承認する

1. 「承認」画面を開く
2. **「承認待ち」タブ** を選択
3. 確認したい投稿の内容を確認
4. **「承認」** ボタンをクリック
  - **予約日時が設定されている場合**: ステータスが「予約中」になり、指定日時に自動投稿されます
  - **予約日時が設定されていない場合**: 即座にXへ投稿され、ステータスが「投稿済み」になります

### 4-2. 投稿を差し戻す

1. 「承認待ち」タブの対象投稿で **「差し戻し」** ボタンをクリック
2. ダイアログが開くので **差し戻し理由** を入力（必須）
3. **「差し戻す」** ボタンをクリック
4. 投稿者が理由を確認して修正・再提出できます

> 差し戻し理由は投稿カードに赤背景で表示されます。

### 4-3. タブの説明


| タブ   | 内容              |
| ---- | --------------- |
| 承認待ち | 承認アクションが必要な投稿一覧 |
| 承認済み | 承認・予約中・投稿済みの投稿  |
| 差し戻し | 差し戻された投稿（理由付き）  |
| 下書き  | まだ承認依頼していない投稿   |


### 4-4. よくある質問

**Q. 承認後もXに投稿されない**
→ アカウントのトークンが期限切れの可能性があります。「アカウント管理」画面で「要トークン更新」を確認し、該当アカウントを再接続してください。

**Q. 「差し戻す」ボタンが押せない（グレーアウトしている）**
→ 差し戻し理由が未入力です。テキストエリアに理由を記入してください。

## 5. 受信箱（インボックス）

**画面**: サイドバー → 「受信箱」

Xアカウントへのメンション・返信・引用・DMを一元管理します。

### 5-1. メッセージを担当者にアサインする

1. 「受信箱」画面を開く
2. 「**未読**」タブで対象のメッセージを確認
3. **「アサイン」** ドロップダウンから担当者を選択
4. ステータスが「対応中」に変わります

### 5-2. 対応を完了にする

1. 対象メッセージの **「完了」** ボタンをクリック
2. ステータスが「完了」に変わり、「完了」タブに移動します

### 5-3. 優先度の見方


| バッジ色  | 優先度 | 内容       |
| ----- | --- | -------- |
| 赤     | 緊急  | 即時対応が必要  |
| オレンジ  | 高   | 優先して対応   |
| グレー   | 通常  | 通常対応     |
| 薄いグレー | 低   | 余裕があれば対応 |


### 5-4. メッセージタイプの見方


| ラベル   | 内容         |
| ----- | ---------- |
| メンション | アカウントへの言及  |
| 返信    | ツイートへの返信   |
| 引用    | ツイートの引用RT  |
| DM    | ダイレクトメッセージ |

## 6. ガードレール

**画面**: サイドバー → 「ガードレール」

> **ルールの有効/無効切り替え**: `admin` ロールのみ

投稿前の自動チェックルールを管理します。ルール違反の投稿は送信がブロックされます。

### 6-1. ルールの種類


| タイプ    | 内容         | 設定例                   |
| ------ | ---------- | --------------------- |
| コンテンツ  | NGワード検出    | 「禁止ワード1」「禁止ワード2」      |
| スケジュール | 深夜帯投稿禁止    | 23時〜6時は投稿不可           |
| リンク    | UTMパラメータ必須 | URLに `utm_source` が必要 |


### 6-2. ルールを有効/無効にする

1. 「ガードレール」画面を開く
2. 対象ルールのスイッチをクリックして切り替え
3. 画面右上に「X / Y ルール有効」と表示されます

> `admin` 以外のロールはスイッチを操作できません。

### 6-3. よくある質問

**Q. 「NGワード〇〇が含まれています」と表示されて投稿できない**
→ 投稿テキストにNGワードが含まれています。対象のワードを削除または言い換えてから再度投稿してください。どのワードがNGかわからない場合は、管理者に確認してください。

**Q. 「深夜帯（23時〜6時）の投稿は禁止されています」と表示される**
→ 予約時間が深夜帯に設定されています。予約日時を変更してください。緊急の場合は管理者にスケジュールルールの一時無効化を依頼してください。

**Q. 「URLにUTMパラメータが付与されていません」と表示される**
→ 投稿内のURLに計測用パラメータが付いていません。URLの末尾に `?utm_source=twitter&utm_medium=social` などを追加してください。

## 7. ソーシャルリスニング

**画面**: サイドバー → 「リスニング」

キーワードを設定して、X上の言及をモニタリングします。

### 7-1. 監視キーワードを追加する

1. 「リスニング」画面を開く
2. テキスト入力欄に **キーワード** を入力
3. **タイプ** を選択：


| タイプ    | 用途             |
| ------ | -------------- |
| 自社     | 自社ブランド名・製品名    |
| ハッシュタグ | キャンペーンハッシュタグなど |
| 競合     | 競合他社名          |
| リスク    | 炎上リスクワード       |
| 業界     | 業界トレンドワード      |
| カスタム   | その他            |


1. **「追加」** ボタンをクリック

### 7-2. センチメントの見方

各キーワードカードに **センチメントバー** が表示されます。

- **緑（ポジティブ）**: 好意的な言及の割合
- **赤（ネガティブ）**: 否定的な言及の割合
- 絵文字：😊 ポジティブ / 😐 ニュートラル / 😞 ネガティブ

### 7-3. キーワードを削除する

キーワードカード右上の **ゴミ箱アイコン** をクリックすると削除されます。

## 8. 分析・レポート

### 8-1. 分析ダッシュボード

**画面**: サイドバー → 「分析」

過去7日間のパフォーマンスを確認します。


| 指標       | 内容                  |
| -------- | ------------------- |
| インプレッション | 投稿の表示回数             |
| エンゲージメント | いいね・RT・返信などの合計      |
| エンゲージ率   | エンゲージメント ÷ インプレッション |
| いいね      | いいね数                |
| リツイート    | リツイート数              |
| クリック     | リンククリック数            |


画面上部の **アカウントドロップダウン** で「全アカウント」または特定アカウントに絞り込めます。

### 8-2. レポートを出力する

**画面**: サイドバー → 「レポート」

1. レポートタイプを選択：


| タイプ        | 内容                    | 出力形式            |
| ---------- | --------------------- | --------------- |
| 週次レポート     | 投稿数・エンゲージ・フォロワー増減     | PDF, CSV        |
| 月次レポート     | KPIトレンド・アカウント比較・担当者実績 | PDF, CSV, Excel |
| キャンペーンレポート | 予算消化・ROI・投稿別成果        | PDF, Excel      |


1. **対象期間**（開始日・終了日）を入力
2. **「レポート生成」** ボタンをクリック

## 12. エラー一覧と対処法

### アカウント接続エラー


| 表示メッセージ                    | 原因                   | 対処法                                |
| -------------------------- | -------------------- | ---------------------------------- |
| X認証がキャンセルされました             | X認証画面で「キャンセル」を押した    | 再度「アカウントを追加」から認証を行う                |
| 無効なOAuthリクエストです            | URLが改ざんされた / セッション切れ | ページを再読み込みして再試行する                   |
| セキュリティエラーが発生しました。再度お試しください | CSRF検証失敗（stateミスマッチ） | ブラウザのCookieを削除して再試行。解消しない場合は管理者へ連絡 |
| トークン取得に失敗しました。再度お試しください    | XのAPIサーバーエラー         | しばらく待ってから再試行。繰り返す場合は管理者へ連絡         |
| Xユーザー情報の取得に失敗しました          | X APIの一時的なエラー        | しばらく待ってから再試行                       |


### 投稿作成エラー


| 表示メッセージ                 | 原因              | 対処法                               |
| ----------------------- | --------------- | --------------------------------- |
| 配信先アカウントを選択してください       | アカウント未選択        | 投稿先アカウントにチェックを入れる                 |
| 投稿テキストを入力してください         | テキスト未入力         | テキスト欄に内容を入力する                     |
| NGワード「〇〇」が含まれています       | ガードレール（NGワード）違反 | 該当ワードを削除・言い換えする                   |
| 深夜帯（23時〜6時）の投稿は禁止されています | 予約時間が深夜帯        | 予約日時を変更する                         |
| URLにUTMパラメータが付与されていません  | UTMパラメータ未付与     | URLに `?utm_source=twitter` 等を追加する |
| 投稿の実行中にエラーが発生しました       | X API呼び出し失敗     | トークンが期限切れの可能性。アカウントを再接続して再試行      |


### 権限エラー


| 状況                       | 原因                     | 対処法                           |
| ------------------------ | ---------------------- | ----------------------------- |
| ページを開くとアカウント管理にリダイレクトされる | admin 専用ページへ権限なしでアクセス  | admin ロールの担当者に対応を依頼する         |
| 「承認」ボタンが表示されない           | ロールが editor または viewer | admin または approver に権限変更を依頼する |
| 「アカウントを追加」ボタンが表示されない     | ロールが admin 以外          | admin に対応を依頼する                |
| 「ガードレール」のスイッチが操作できない     | ロールが admin 以外          | admin に設定変更を依頼する              |
