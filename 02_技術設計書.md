### 2.2 データフロー

#### 投稿ライフサイクル

```
下書き作成
  ↓
ガードレール自動チェック（NGワード・画像サイズ・UTM等）
  ↓  NG → エラー表示・修正要求
承認依頼
  ↓
承認者に通知（メール / Slack）
  ↓
承認 → 予約キュー登録    差し戻し → 下書きに戻る（理由付き）
  ↓
Vercel Cron（1分間隔）で scheduled_at をチェック
  ↓
X API v2 POST /2/tweets 実行
  ↓
成功 → ステータス更新 + 監査ログ記録
失敗 → リトライキュー + アラート通知
```

#### 受信箱データフロー

```
X API Webhook or Polling（5分間隔）
  ↓
受信データ正規化（メンション/返信/引用/DM を統一フォーマット）
  ↓
DB保存（inbox_items テーブル）
  ↓
優先度自動判定（キーワードマッチ・フォロワー数等のルール）
  ↓
ダッシュボード表示
  ↓
担当者アサイン → 対応 → 完了マーク
```

#### リスニングデータフロー

```
Vercel Cron（15分間隔）
  ↓
X API v2 GET /2/tweets/search/recent（キーワードごと）
  ↓
センチメント分析（テキスト分類）
  ↓
DB保存（listening_mentions テーブル）
  ↓
閾値チェック → 超過時にアラート通知
```

## 3. データベース設計

### 3.1 ER概要（主要テーブル）

```

               └─── audit_logs
               
x_accounts ──── inbox_items
```

### 3.2 テーブル定義

#### organizations（組織）

```sql
CREATE TABLE organizations (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name          VARCHAR(255) NOT NULL,
  plan          VARCHAR(50) NOT NULL DEFAULT 'free',
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

#### users（ユーザー）

```sql
CREATE TABLE users (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id        UUID NOT NULL REFERENCES organizations(id),
  email         VARCHAR(255) NOT NULL UNIQUE,
  name          VARCHAR(255) NOT NULL,
  role          VARCHAR(20) NOT NULL DEFAULT 'viewer',
  -- role: 'admin' | 'approver' | 'editor' | 'viewer'
  avatar_url    TEXT,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

#### x_accounts（Xアカウント接続情報）

```sql
CREATE TABLE x_accounts (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id            UUID NOT NULL REFERENCES organizations(id),
  x_user_id         VARCHAR(50) NOT NULL UNIQUE,
  handle            VARCHAR(50) NOT NULL,
  name              VARCHAR(255) NOT NULL,
  avatar_url        TEXT,
  verified          BOOLEAN NOT NULL DEFAULT FALSE,
  category          VARCHAR(50),
  status            VARCHAR(20) NOT NULL DEFAULT 'active',
  -- status: 'active' | 'paused'
  token_encrypted   BYTEA NOT NULL,
  token_expires_at  TIMESTAMPTZ NOT NULL,
  followers_count   INTEGER NOT NULL DEFAULT 0,
  following_count   INTEGER NOT NULL DEFAULT 0,
  posts_count       INTEGER NOT NULL DEFAULT 0,
  last_post_at      TIMESTAMPTZ,
  created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

#### user_account_permissions（ユーザー×アカウント権限）

```sql
CREATE TABLE user_account_permissions (
  user_id       UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  x_account_id  UUID NOT NULL REFERENCES x_accounts(id) ON DELETE CASCADE,
  role          VARCHAR(20) NOT NULL DEFAULT 'viewer',
  -- role: 'admin' | 'approver' | 'editor' | 'viewer'
  PRIMARY KEY (user_id, x_account_id)
);
```

#### posts（投稿マスター）

```sql
CREATE TABLE posts (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id        UUID NOT NULL REFERENCES organizations(id),
  status        VARCHAR(20) NOT NULL DEFAULT 'draft',
  -- status: 'draft' | 'pending' | 'approved' | 'rejected' | 'scheduled' | 'published' | 'failed'
  scheduled_at  TIMESTAMPTZ,
  published_at  TIMESTAMPTZ,
  author_id     UUID NOT NULL REFERENCES users(id),
  approver_id   UUID REFERENCES users(id),
  reject_reason TEXT,
  campaign_id   UUID REFERENCES campaigns(id),
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_posts_org_status_scheduled 
  ON posts(org_id, status, scheduled_at);
```

#### post_variants（アカウント別投稿文面）

```sql
CREATE TABLE post_variants (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  post_id       UUID NOT NULL REFERENCES posts(id) ON DELETE CASCADE,
  x_account_id  UUID NOT NULL REFERENCES x_accounts(id),
  text          TEXT NOT NULL,
  media_urls    JSONB DEFAULT '[]',
  x_tweet_id    VARCHAR(50),
  -- 投稿成功時にXのtweet IDを記録
  UNIQUE(post_id, x_account_id)
);
```

#### inbox_items（受信箱）

```sql
CREATE TABLE inbox_items (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  x_account_id  UUID NOT NULL REFERENCES x_accounts(id),
  type          VARCHAR(20) NOT NULL,
  -- type: 'mention' | 'reply' | 'quote' | 'dm'
  from_handle   VARCHAR(50) NOT NULL,
  from_user_id  VARCHAR(50),
  text          TEXT NOT NULL,
  x_tweet_id    VARCHAR(50),
  status        VARCHAR(20) NOT NULL DEFAULT 'unread',
  -- status: 'unread' | 'assigned' | 'done'
  assignee_id   UUID REFERENCES users(id),
  priority      VARCHAR(10) NOT NULL DEFAULT 'normal',
  -- priority: 'urgent' | 'high' | 'normal' | 'low'
  received_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_inbox_account_status 
  ON inbox_items(x_account_id, status);
```

#### analytics_snapshots（日次分析スナップショット）

```sql
CREATE TABLE analytics_snapshots (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  x_account_id  UUID NOT NULL REFERENCES x_accounts(id),
  date          DATE NOT NULL,
  impressions   BIGINT NOT NULL DEFAULT 0,
  engagements   INTEGER NOT NULL DEFAULT 0,
  likes         INTEGER NOT NULL DEFAULT 0,
  retweets      INTEGER NOT NULL DEFAULT 0,
  replies       INTEGER NOT NULL DEFAULT 0,
  clicks        INTEGER NOT NULL DEFAULT 0,
  followers     INTEGER NOT NULL DEFAULT 0,
  following     INTEGER NOT NULL DEFAULT 0,
  UNIQUE(x_account_id, date)
);

CREATE INDEX idx_analytics_account_date 
  ON analytics_snapshots(x_account_id, date);
```

#### listening_keywords（監視キーワード）

```sql
CREATE TABLE listening_keywords (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id          UUID NOT NULL REFERENCES organizations(id),
  keyword         VARCHAR(255) NOT NULL,
  type            VARCHAR(20) NOT NULL DEFAULT 'custom',
  -- type: 'brand' | 'hashtag' | 'competitor' | 'risk' | 'industry' | 'custom'
  alert_threshold INTEGER DEFAULT 10,
  enabled         BOOLEAN NOT NULL DEFAULT TRUE,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

#### listening_mentions（検出履歴）

```sql
CREATE TABLE listening_mentions (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  keyword_id    UUID NOT NULL REFERENCES listening_keywords(id) ON DELETE CASCADE,
  x_tweet_id    VARCHAR(50) NOT NULL,
  text          TEXT NOT NULL,
  sentiment     REAL NOT NULL DEFAULT 0.5,
  -- 0.0 = very negative, 1.0 = very positive
  author_handle VARCHAR(50),
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_listening_mentions_keyword_date 
  ON listening_mentions(keyword_id, created_at);
```

#### guardrails（ガードレール設定）

```sql
CREATE TABLE guardrails (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id      UUID NOT NULL REFERENCES organizations(id),
  name        VARCHAR(255) NOT NULL,
  type        VARCHAR(20) NOT NULL,
  -- type: 'content' | 'media' | 'link' | 'schedule' | 'legal'
  config_json JSONB NOT NULL DEFAULT '{}',
  enabled     BOOLEAN NOT NULL DEFAULT TRUE,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

#### campaigns（キャンペーン）

```sql
CREATE TABLE campaigns (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id      UUID NOT NULL REFERENCES organizations(id),
  name        VARCHAR(255) NOT NULL,
  status      VARCHAR(20) NOT NULL DEFAULT 'planned',
  -- status: 'planned' | 'active' | 'completed'
  start_date  DATE NOT NULL,
  end_date    DATE NOT NULL,
```

#### campaign_accounts（キャンペーン×アカウント）


#### audit_logs（監査ログ）

```sql
CREATE TABLE audit_logs (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id      UUID NOT NULL REFERENCES organizations(id),
  user_id     UUID REFERENCES users(id),
  action      VARCHAR(50) NOT NULL,
  target_type VARCHAR(50) NOT NULL,
```

### 4.2 X API v2連携

X API v2の以下のエンドポイントを使用する。Rate LimitはUpstash Redisでトークンバケット制御を行う。

| 用途 | X APIエンドポイント | Rate Limit |
|---|---|---|
| 投稿 | `POST /2/tweets` | 200 req / 15min / user |
| 削除 | `DELETE /2/tweets/:id` | 50 req / 15min / user |
| メンション取得 | `GET /2/users/:id/mentions` | 180 req / 15min / user |
| ユーザー情報 | `GET /2/users/:id` | 300 req / 15min / app |
| DM取得 | `GET /2/dm_events` | 300 req / 15min / user |
| 検索（リスニング） | `GET /2/tweets/search/recent` | 450 req / 15min / app |
| メディアアップロード | `POST /1.1/media/upload` | チャンク方式 |

#### Rate Limit制御

```typescript
// lib/x-api/rate-limiter.ts
import { Ratelimit } from "@upstash/ratelimit";
import { Redis } from "@upstash/redis";

const redis = Redis.fromEnv();

export const tweetRateLimit = new Ratelimit({
  redis,
  limiter: Ratelimit.slidingWindow(200, "15 m"),
  prefix: "x-api:tweet",
});

export const searchRateLimit = new Ratelimit({
  redis,
  limiter: Ratelimit.slidingWindow(450, "15 m"),
  prefix: "x-api:search",
});
```

### 4.3 認証フロー

```
1. ユーザーが「アカウントを追加」をクリック
2. サーバーがOAuth 2.0 PKCE認証URLを生成
   - code_verifier をセッションに保存
   - X OAuth authorize URLにリダイレクト
3. ユーザーがX側で認可
4. コールバックURLにauthorization_codeが返る
5. サーバーがcode + code_verifierでトークン交換
6. access_token + refresh_tokenをAES-256-GCMで暗号化してDB保存
7. x_accountsテーブルにアカウント情報を登録
8. ダッシュボードにリダイレクト
```

### 6.2 Dockerfile（マルチステージ）

```dockerfile
# ── Stage 1: Development ──
FROM node:20-alpine AS development
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci
COPY . .
RUN npx prisma generate
EXPOSE 3000
CMD ["npm", "run", "dev"]

# ── Stage 2: Dependencies ──
FROM node:20-alpine AS deps
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci --omit=dev

# ── Stage 3: Builder ──
FROM node:20-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npx prisma generate
RUN npm run build

# ── Stage 4: Runner (Production) ──
FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production
RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/prisma ./prisma

USER nextjs
EXPOSE 3000
ENV PORT=3000
ENV HOSTNAME="0.0.0.0"
CMD ["node", "server.js"]
```

### 6.3 ローカル開発手順

```bash
# 1. リポジトリをクローン
git clone https://github.com/your-org/x-account-manager.git

# 2. 環境変数を設定

# .env.local に X_CLIENT_ID, X_CLIENT_SECRET 等を記入

# 3. Docker起動
docker compose up -d

# 4. DBマイグレーション
docker compose exec app npx prisma migrate dev

# 5. シードデータ投入（任意）
docker compose exec app npx prisma db seed

# 6. ブラウザで開く
open http://localhost:3000

# ── よく使うコマンド ──
docker compose logs -f app          # ログ確認
docker compose exec app npx prisma studio  # DB GUI
docker compose down                  # 停止
docker compose down -v               # 停止 + データ削除
```
