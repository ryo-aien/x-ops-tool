## 1. プロジェクト概要

### 1.1 背景と目的

複数のX（旧Twitter）アカウントを運用する企業・代理店・個人向けに、一元管理ダッシュボードを提供する。X公式APIと連携し、投稿管理・分析・カスタマーサポート・リスク管理を包括的にカバーするプロダクトを開発する。

### 1.2 ターゲットユーザー

| ユーザータイプ | 主なニーズ | 例 |
|---|---|---|
| マーケティング担当者 | 複数ブランドの投稿管理・予約・分析 | 事業会社SNS担当 |
| カスタマーサポート | 返信・DMの一元管理・アサイン | ECサポートチーム |
| ブランド責任者 | 承認フロー・リスク監視・レポート | マーケティングマネージャー |
| 代理店 | 複数クライアントの横断管理・レポート出力 | SNS運用代行会社 |

### 1.3 スコープ

**MVP スコープ（v1.0）:** 以下10機能を実装対象とする。各機能の詳細は第2章以降で定義する。

| # | 機能名 | 優先度 |
|---|---|---|
| F1 | アカウント切替・一覧管理 | Must |
| F2 | 投稿作成・一括予約投稿 | Must |
| F3 | 統合受信箱 | Must |
| F4 | 投稿承認ワークフロー | Must |
| F5 | 分析ダッシュボード | Must |
| F6 | ソーシャルリスニング | Should |
| F7 | 権限管理・監査ログ | Must |
| F8 | ブランド運用ガードレール | Should |
| F9 | キャンペーン運用補助 | Could |
| F10 | レポート出力 | Should |

## 2. 機能仕様

### 2.1 F1: アカウント切替・一覧管理

運用担当者のホーム画面。全アカウントのステータスを一覧し、ワンクリックでフィルタ切替できる。

| 項目 | 説明 |
|---|---|
| プロフィール表示 | アイコン・名前・ハンドル・認証バッジ・カテゴリタグ |
| 権限状態 | 管理者 / 承認者 / 投稿者 / 閲覧者の4段階ロール表示 |
| 接続状態 | OAuthトークンの有効/期限切れ/未接続をドット表示 |
| 最終投稿日時 | 最後に公開された投稿のタイムスタンプ |
| 未返信件数 | 未対応のメンション・リプライ・DMの合計 |
| フォロワー数推移 | ミニスパークラインチャートで直近7日間を表示 |
| アカウント追加 | OAuth 2.0 PKCEフローで新規接続 |
| アカウント一時停止 | トークン保持のまま投稿・受信を無効化 |

**画面イメージ:**
- サマリー行: 総フォロワー / アクティブ数 / 未返信合計 / 要トークン更新
- アカウントカード: クリックで全画面フィルタ切替

---

### 2.2 F2: 投稿作成・一括予約投稿

下書き作成から承認・予約までの投稿ライフサイクルを管理する。複数アカウントへの同時配信が可能。

| 機能 | 説明 |
|---|---|
| 複数アカウント同時配信 | チェックボックスで配信先を複数選択 |
| アカウント別文面差替 | トグルONでアカウントごとに個別テキストを編集 |
| 画像・動画添付 | ドラッグ＆ドロップまたはファイルピッカーでメディア添付 |
| 予約投稿 | 日時指定でキューに登録、cronで自動投稿 |
| 下書き保存 | 自動保存または手動保存でドラフト保持 |
| 文字数カウント | 280文字制限をリアルタイム表示（超過時に赤色警告） |
| ガードレール連携 | 投稿前にNGワード・画像サイズ・UTM等を自動チェック |
| 承認依頼 | 「承認依頼」ボタンで承認者に通知送信 |

**投稿ステータス遷移:**

```
下書き → 承認待ち → 承認済み → 予約キュー → 投稿完了
                  ↘ 差し戻し → 下書き（修正）
```

---

### 2.3 F3: 統合受信箱

各アカウント宛のメンション・返信・引用・DMを統合的に表示し、担当者アサインでカスタマーサポートを効率化する。

| 機能 | 説明 |
|---|---|
| タイプ別フィルタ | メンション / 返信 / 引用 / DM |
| ステータス管理 | 未読 → アサイン済 → 対応完了 |
| 優先度設定 | 緊急 / 高 / 通常 / 低の4段階 |
| 担当者アサイン | ドロップダウンでチームメンバーに割り当て |
| アカウント横断検索 | 全アカウントまたは特定アカウントで絞り込み |
| 緊急アラート | 優先度「緊急」のアイテムはパルスアニメーション+リスト最上位 |
| DM注意事項 | X API v2のDM権限は別途アプリ権限申請が必要 |

**サマリーカード:**
- 未読数 / 対応中数 / 完了数 / 緊急数を常時表示

---

### 2.4 F4: 投稿承認ワークフロー

事故防止のための承認プロセス。下書き → 承認待ち → 承認済み / 差し戻しのステートマシンを実装する。

| 機能 | 説明 |
|---|---|
| ステート管理 | 下書き → 承認待ち → 承認済み / 差し戻し |
| 承認者指定 | アカウントごとに承認権限者を設定 |
| 差し戻し理由 | 承認者がコメント付きで修正依頼可能 |
| ルールエンジン | 深夜投稿禁止・URL必須・法務チェック等を自動判定 |
| 通知 | 承認待ち発生時にメール / Slack通知 |
| 複数アカウント投稿の承認 | 1投稿に複数アカウント分の文面がある場合、全文面をまとめて表示 |
| 履歴 | 承認/却下の全履歴を監査ログに記録 |

**承認画面のタブ構成:**
- 承認待ち / 承認済み / 差し戻し / 下書き（件数バッジ付き）

---

### 2.5 F5: 分析ダッシュボード

アカウント単体および横断比較で、KPIを可視化する。

| 指標 | 説明 | 表示形式 |
|---|---|---|
| インプレッション | 投稿の表示回数 | 棒グラフ + 数値 |
| エンゲージメント | いいね・Rト・リプライの合計 | 棒グラフ + 数値 |
| エンゲージ率 | エンゲージメント / インプレッション × 100 | パーセント表示 |
| フォロワー増減 | 期間内の純増減 | 数値 + トレンド矢印 |
| 返信率 | 返信数 / インプレッション × 100 | パーセント表示 |
| クリック数 | リンククリック数 | 数値 |
| トップ投稿 | インプレッション順ランキング | リスト表示 |
| 横断比較テーブル | 全アカウントのKPIを一覧表示 | テーブル |

**表示モード:**
- **横断比較モード:** 全アカウントのKPIを1テーブルで比較 + アカウント別小チャート
- **個別モード:** 特定アカウントの詳細分析（チャート + トップ投稿）

---

### 2.6 F6: ソーシャルリスニング

自社名・競合・製品・ハッシュタグを監視し、炎上兆候を早期検知する。

| 機能 | 説明 |
|---|---|
| キーワード登録 | 自社・競合・ハッシュタグ・リスクワードを登録 |
| メンション数集計 | 過去24h / 7日 / 30日の言及数推移 |
| センチメント分析 | ポジティブ/ニュートラル/ネガティブの割合をバー表示 |
| トレンド検知 | 急激なメンション増やネガティブ急上昇をアラート化 |
| 炎上アラート | 閾値超過時にSlack / メールで即時通知 |
| キーワード種別 | 自社 / ハッシュタグ / 競合 / リスク / 業界 / カスタム |

**アラート条件の例:**
- リスクキーワードのセンチメントが30%以下かつメンション数が前日比200%以上

### 2.7 F7: 権限管理・監査ログ

誰がどのアカウントで何をしたかを全て記録し、企業ガバナンスを実現する。

**チームメンバー管理:**

| 項目 | 説明 |
|---|---|
| ロール | 管理者 / 承認者 / 投稿者 / 閲覧者 |
| アクセス範囲 | ユーザーごとにアクセス可能なアカウントを指定 |
| 招待 | メールアドレスで新メンバーを招待 |

**監査ログ対象操作:**

| 対象操作 | 記録内容 |
|---|---|
| 投稿操作 | 作成・編集・削除・公開・予約 |
| 承認操作 | 承認・却下・差し戻し |
| アカウント操作 | 接続・切断・トークン更新 |
| 権限操作 | ロール変更・メンバー追加/削除 |
| システム操作 | トークン自動更新・ガードレール変更 |

**ログ機能:**
- 全文検索・フィルタ対応
- CSVエクスポート

### 2.8 F8: ブランド運用ガードレール

「投稿できる」より「事故らない」を重視した自動チェック機能。

| ルール名 | 種別 | 説明 |
|---|---|---|
| NGワードフィルタ | コンテンツ | 登録語句を含む投稿をブロック |
| 禁止リンク検出 | コンテンツ | ブラックリストドメインへのリンクを検出 |
| 画像サイズチェック | メディア | 最小600x335px・最大5MBを検証 |
| UTM自動付与 | リンク | URLにutm_source=x等を自動追加 |
| 深夜投稿禁止 | スケジュール | 23:00〜06:00の投稿をブロック |
| URL短縮必須 | リンク | bit.ly等での短縮を強制 |
| 法務テンプレート差込 | 法務 | キャンペーン投稿に免責文言を自動付加 |
| トーンチェック（AI） | コンテンツ | ブランドガイドラインへの準拠度をAI判定 |

**設定:**
- 各ルールは個別にON/OFF切替可能
- カスタムルール追加に対応

---

### 2.9 F9: キャンペーン運用補助

広告アカウント・キャンペーンごとに、オーガニック投稿との整合を管理する。

| 機能 | 説明 |
|---|---|
| キャンペーンCRUD | 名前・期間・対象アカウント・予算を設定 |
| 予算消化率 | 消化額 / 予算のプログレスバー |
| 投稿紐付け | オーガニック / 広告投稿をキャンペーンに紐付け |
| KPIトラッキング | インプレッション / エンゲージ / CVR |
| ステータス管理 | 計画中 / 実施中 / 完了 |

---

### 2.10 F10: レポート出力

週次・月次・キャンペーン別のレポートを自動生成し、PDF / CSV / Excelで出力する。

| レポート種別 | 含まれる内容 | 出力形式 |
|---|---|---|
| 週次レポート | 投稿数・エンゲージ・フォロワー増減・トップ投稿 | PDF / CSV |
| 月次レポート | KPIトレンド・アカウント比較・担当者別実績 | PDF / CSV / Excel |
| キャンペーンレポート | 予算消化・ROI・投稿別成果・前回比較 | PDF / Excel |

**フィルタ:**
- 対象アカウント / 担当者 / 期間で絞り込み可能

**レポート履歴:**
- 過去に生成したレポートの一覧・再ダウンロード対応

## 3. 非機能要件

| カテゴリ | 要件 | 目標値 |
|---|---|---|
| パフォーマンス | ページ初期表示時間 | LCP < 2.5秒 |
| パフォーマンス | APIレスポンス時間 | p95 < 500ms |
| 可用性 | サービス稼働率 | 99.9% SLA（Vercel依存） |
| セキュリティ | 認証方式 | NextAuth.js + OAuth 2.0 PKCE |
| セキュリティ | トークン保管 | AES-256暗号化でDB保存 |
| セキュリティ | RBAC | ロールベースアクセス制御を全APIに適用 |
| スケーラビリティ | 同時管理アカウント数 | 初期50アカウントまで |
| 国際化 | 対応言語 | 日本語（初期）、英語（v1.1） |

## 4. 用語集

| 用語 | 説明 |
|---|---|
| X API v2 | X（旧Twitter）が提供するREST API。OAuth 2.0 PKCE認証に対応 |
| OAuth 2.0 PKCE | Proof Key for Code Exchange。モバイル/SPAに適した安全なOAuth認証フロー |
| RBAC | Role-Based Access Control。ロールに基づくアクセス制御 |
| UTM | Urchin Tracking Module。URLパラメータによる流入元トラッキング |
| センチメント | テキストの感情傾向（ポジティブ/ネガティブ/ニュートラル）の数値化 |
| ガードレール | 投稿前に自動チェックされる安全ルールの総称 |
| CVR | Conversion Rate。コンバージョン率 |
